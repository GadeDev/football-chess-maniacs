## 1. 設計意図（最重要・ここに立ち返る）

### 1.1 中心思想：タイトルが対戦の入口そのもの

Krunker の「軽さ」の正体は「タイトル＝即プレイ」であること。
タイトル画面を単なる入口ではなく、**速い層・楽しさ・全体像が一望できるハブ**にする。

### 1.2 遊びやすさと楽しさは本来トレードオフ

- 遊びやすさを突き詰める → 画面を削り編成を飛ばす → だが編成は本来楽しい部分なので、削りすぎると楽しさが死ぬ
- 楽しさを演出する → 高揚感のある遷移を足す → だが手数が増えて遊びやすさが落ちる

**両立の鍵は「初回」と「2回目以降」を分けること。**

### 1.3 2層構造

| 層 | 役割 | 担保するもの |
|----|------|------------|
| 速い層 | 「前回の編成で対戦」をタイトル最上段に置き、前回設定を記憶してワンタップでマッチングへ | 遊びやすさ（実質2タップ） |
| こだわる層 | 編成（旧 Formation）を試合フローの必須通過点から外し、「いじりたい時に入る楽しむ場」に格上げ | 楽しさ |

> 注意: 編成を「スキップ対象の面倒なもの」扱いにしない。
> **速い層を主役にして、編成は寄り道にする**という発想の転換。

## 2. 現状の動線（整理前）

| モード | 画面遷移 | 試合前の画面数 |
|--------|----------|----------------|
| COM対戦 | Title → ModeSelect → DifficultySelect → TeamSelect → Formation → Matching → Battle | 6 |
| ランク/カジュアル | Title → ModeSelect → TeamSelect → Formation → Matching → Battle | 5 |
| COM観戦 | Title → ModeSelect → Matching → Battle | 3 |

### 洗練を妨げている主因

1. **TeamSelect が形骸化**（112行）— 2チームのモックだけで実質「次へ」を押すだけ。Formation と役割が重複
2. **毎回フル編成を強制**— 一番遊ばれる COM 対戦でも毎試合 985行の Formation を通る。「前回と同じ編成でサッと」の導線がない
3. **DifficultySelect だけ別画面**（80行）— COM のときだけ画面が1枚増える不整合
4. **設定が4画面に分散**— モード・難易度・チーム・編成がバラバラで全体像が見えない
5. **演出がなく事務的**— 「これから試合だ」という高揚感がない

## 3. 整理後の動線（正本）

```
[タイトル（ハブ化）]
   ├─「前回の編成で対戦」──────────────▶ [マッチング] → 試合   ← 最速・主役
   ├─「モードを選んで対戦」──▶ [対戦セットアップ]
   │                              ├─「編成して開始」──▶ [編成]
   │                              └─「この設定で開始」──▶ [マッチング]
   └─「編成をいじる」────────────────▶ [編成] →「この編成で対戦」▶ [マッチング]

[リザルト] ─┬─「もう一度」────▶ [マッチング]
            ├─「編成を変える」─▶ [編成]
            └─「タイトルへ」──▶ [タイトル]
```

### 変更点サマリ

| 変更 | 内容 | 効果 |
|------|------|------|
| 「前回の編成で対戦」追加 | モード・難易度・チーム・編成を保存し、タイトルから Matching へ直行 | 遊びやすさ↑（2回目以降は実質2タップ） |
| TeamSelect 廃止 | Formation に吸収（チーム＝編成そのもの） | わかりやすさ↑・手数↓ |
| DifficultySelect 統合 | 対戦セットアップ画面内に難易度を展開（COM選択時のみ） | 別画面の不整合を解消 |
| Formation を任意ステップ化 | 試合フローの必須通過点から外し、タイトル/セットアップ/リザルトの3箇所から入れる「楽しむ場」へ | 楽しさ↑ |
| キックオフ演出 | マッチング→試合の間に入場・スコアボード点灯等 | 楽しさ↑ |

### 整理後の最短手数

- COM対戦（2回目以降）: **タイトル →（前回の編成で）マッチング → 試合**＝実質2タップ

## 4. 実装計画（Claude Code 作業単位）

> 既存はフルスペック。各タスクは原則「既存画面の削除・統合・配線変更」。
> 対象ファイルの行数は現状把握用の目安。

### Phase 1: 速い層（遊びやすさに直結・最優先）

**T1. 前回設定の永続化**
- 保存対象: モード / 難易度 / 選択チーム / 編成内容（スタメン・配置・ベンチ）
- 現状: ロスターは状態管理のみで再起動で消える前提 → 永続化層を追加
- 受け入れ条件: 一度対戦を開始した後、再起動しても前回設定が復元される

**T2. 「前回の編成で対戦」ボタン追加**
- タイトル最上段に最大サイズで配置。サブラベルに前回設定（例「レギュラー · メインチーム」）を表示
- 前回設定が無い初回は非表示 or 「モードを選んで対戦」にフォールバック
- 受け入れ条件: タイトルからワンタップで Matching に直行し、前回と同じ条件で試合が始まる

### Phase 2: 画面の統合・削除（わかりやすさ・手数削減）

**T3. TeamSelect 廃止 → Formation へ吸収**
- TeamSelect（112行）を導線から除去。チーム選択を Formation 内に統合
- 注記: TeamSelect は現状 API 未接続のモック。吸収時に本来のチーム選択 UI を Formation に持たせる
- 受け入れ条件: TeamSelect を経由せずチームを選んで編成できる

**T4. DifficultySelect 統合 → 対戦セットアップ画面**
- DifficultySelect（80行）を独立画面から廃止し、ModeSelect を「対戦セットアップ」に拡張
- COM 選択時のみ同一画面内に難易度（ビギナー/レギュラー/マニアック）を展開
- 受け入れ条件: COM だけ画面が増える不整合が解消され、1画面でモード＋難易度が決まる

**T5. Formation を任意ステップ化**
- 試合フローの必須通過点から外す。タイトル / 対戦セットアップ / リザルトの3箇所から入れる
- Formation 内に「この編成で対戦」を置き、ここで完結してマッチングへ行けるように
- 受け入れ条件: 編成を通らずに対戦開始でき、かつ任意のタイミングで編成に入れる

### Phase 3: 楽しさの演出

**T6. キックオフ演出**
- マッチング→試合の間に入場・スコアボード点灯等の演出を挿入
- 受け入れ条件: 試合開始前に高揚感のある演出が再生される（スキップ可が望ましい）

**T7. タイトルの第一印象**
- 背景でコマ/盤面が動く等、起動直後に「楽しそう」が伝わる演出

## 6. 用語

- **アウトゲーム**: 試合前の流れ（タイトル〜編成〜マッチング）
- **速い層 / こだわる層**: §1.3 の2層構造
- **ハブ**: 速い層・楽しさ・全体像が集約されたタイトル画面
