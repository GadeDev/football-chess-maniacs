# 編成画面 仕様書 v3

> v3改訂（2026-07-07、正本の決定記録: `docs/formation_review_20260707.md`）:
> ①所持コマをサーバー実データ（`/api/pieces`）に接続 ②セーブ/ロードをサーバー保存（`/api/teams`+entitlement枠）に変更 ③「保存」と「対戦」を分離 ④カードを実キャラ名+era表示に統一 ⑤初期所持をFounding Eleven（サーバー実装）に合わせて改訂

## コンセプト

プレイヤーは**所有しているコマ（最大200個）の中から選んで配置**する。
コマはショップ（別画面）で購入して集める。
各コマにはポジションとコストが固定されており、編成画面では変更できない。

## データソース（v3）

| データ | ログインユーザー | ゲスト |
|---|---|---|
| 手持ちコマ | `POST /api/pieces/sync`（Founding Eleven付与+Platform entitlement同期）→ `GET /api/pieces`（実キャラ: name_ja/name_en・era・position・cost） | Founding Eleven 11キャラをカタログ（`GET /api/shop/catalog`、公開API）から解決。取得失敗時はローカル定義でフォールバック |
| セーブスロット | `GET/POST/PUT/DELETE /api/teams`（D1、スロット1〜10） | なし（編成ドラフトをlocalStorageに自動保存のみ） |
| 編成の対戦反映 | `FormationData`（クライアント直接）+ オンラインは D1 `teams` の is_active チーム | `FormationData`（クライアント直接、COM対戦のみ） |

## コマの基本ルール

### 所有
- コマは最大200個まで所有可能
- 同じ種類（同ポジション・同コスト）のコマを複数所有できる
- 各コマに名前をつけられる

### コマの種類
ポジション8種 × コスト5段階 ＝ **40種類**

| ポジション | コスト1 | コスト1.5 | コスト2 | コスト2.5 | コスト3 |
|---|---|---|---|---|---|
| GK | ○ | ○ | ○ | ○ | ○ |
| DF | ○ | ○ | ○ | ○ | ○ |
| SB | ○ | ○ | ○ | ○ | ○ |
| VO | ○ | ○ | ○ | ○ | ○ |
| MF | ○ | ○ | ○ | ○ | ○ |
| OM | ○ | ○ | ○ | ○ | ○ |
| WG | ○ | ○ | ○ | ○ | ○ |
| FW | ○ | ○ | ○ | ○ | ○ |

### チーム編成ルール
- **スタメン：11枚**（GK 1人 + フィールドプレイヤー 10人）
- **スタメン合計コスト上限：16**
- **サブ（ベンチ）：9枚**（コスト制限なし）
- **合計：20枚**で1チーム
- GK以外のポジション制限なし（全員FWでもOK）
- GKは1人固定

### 試合中の選手交代ルール
- **交代機会：3回**（1回に複数人交代OK）
- **交代人数上限：合計5人まで**（サッカー準拠）
- 交代後も**ピッチ内の合計コストは16以内**であること
- サブからスタメンと同コスト以上のコマを出す場合、スタメン側を下げてコスト枠を確保する

### ランク帯システム（コスト判定の特殊ルール）

コストは3つのランク帯に分類される：

| ランク帯 | 含まれるコスト |
|---|---|
| 低ランク帯 | 1, 1.5 |
| 中ランク帯 | 2, 2.5 |
| 高ランク帯 | 3 |

コマ同士が対峙する判定（タックル・パスカット・シュート等）では、生のコスト差ではなく「有効差」を使用する：

| 対峙パターン | 例 | 有効差 |
|---|---|---|
| 同コスト | 1vs1, 2vs2, 3vs3 | 0 |
| 異ランク帯 | 1vs2, 1vs3, 2vs3 等 | 1 |
| 同ランク帯の0.5差 | 1vs1.5, 2vs2.5 | 2（最大） |

※0.5の差が同ランク帯のミラーマッチで最大効果を発揮する設計。

初回同期時に以下の実在キャラ11人（全コスト1）が自動付与される:

| piece_id | ポジション | キャラクター |
|---|---|---|
| 8 | GK | トム・ハーディング |
| 9 | DF | イライジャ・マッケイ |
| 55 | DF | マリウス・ベックマン |
| 37 | SB | ヨゼフ・ハートマン |
| 70 | SB | エルネスト・リベラ |
| 35 | VO | ルーシー・ブライス |
| 10 | MF | サミュエル・リード |
| 82 | MF | ケヴィン・マホーニー |

※初期はスタメン11枚のみ。ベンチなし（初心者に選手交代をさせないため）。
※**OMのみ初期所持なし**。ショップで購入して入手する。
※ベンチ枠のコマはショップで買い足していく。
※ゲストも同じ11キャラを使用する（ローカル解決、購入・保存は不可）。

## 画面構成

### ヘッダー部
- クラブ名・監督名
- ユーザーID
- 総コスト表示：「コスト XX / 16」（リアルタイム更新）

### セーブスロット（v3: サーバー保存）
- 最大10スロット（番号1〜10固定）
- 各スロットに名前をつけられる（任意、サーバー `teams.name`）
- **利用可能枠数はentitlementで決まる**（サーバー `/api/teams` が判定・返却）:
  - 無料: 1枠
  - セーブ枠買い切り `fcms_save_slots_9`: 10枠
  - サブスク `uf_subscription_monthly_premium`: 1+3枠
  - 期限切れで枠数が減った場合、上位スロットは**読み取り専用**（ロード可・上書き不可）
- ゲスト: セーブスロットUIは表示するがセーブ操作で`requireLogin`のログイン誘導。編成ドラフトはlocalStorageに自動保存（リロードで消えない）

### フィールド部（メインエリア）
- サッカーフィールド風のビジュアル
- スタメン11枚のコマがフィールド上に配置される
- コマは**フィールド上の自由な位置に配置可能**（自由配置）
- 各コマの表示内容：
  - ポジション略称（GK, DF, SB, VO, MF, OM, WG, FW）
  - コスト数値
  - 名前（設定済みの場合）
  - ポジションごとの色分け

### サブエリア
- フィールド下部にベンチ9枠を表示
- サブのコマを並べて表示

### 手持ちコマ一覧（カードグリッド）

スタメンやサブのコマを入れ替えるときに表示される。所有コマをカードグリッド形式（2〜3列）で一覧表示する。

**カードの表示内容（v3: 実キャラ統一）：**
- コマトークン画像（`public/assets/pieces/`、ランク別。顔イラストは素材が揃い次第差し替え）
- ポジション略称（左上バッジ）
- コスト★マーク（右上）
- **実キャラクター名**（下部。`name_ja`/ロケール別、CollectionScreenと同一）
- **時代（Era 1〜7）表示**（CollectionScreenの棚と対応）
- コスト枠装飾（1=無装飾, 1.5=銅枠, 2=銀枠, 2.5=金枠, 3=虹枠）※素材が揃い次第

**カードの状態表示：**
- 使用中のコマ → 半透明 +「使用中」ラベル（選択不可）
- コスト16超過になるコマ → グレーアウト（選択不可）
- 選択可能なコマ → 通常表示、タップで即配置

**フィルター・ソート（画面上部）：**
- ポジションフィルター：アイコンボタン8つ + 「全部」
- コスト順ソート：昇順/降順切り替えボタン

### フッター部（v3.1: 主行動1つの3ボタン構成 — 2026-07-08改訂）
- ガイド文（1行・小）: 「保存するとこの編成が以後のすべての対戦で使われます」
- 「← マイページ」（控えめ）→ タイトルへ戻る（ドラフト自動保存があるため編成は失われない）
- 「💾 保存」(青) → その場でスロット保存（画面に留まる）。ゲストは requireLogin ログイン誘導
- **主ボタン（緑・大・常設）**:
  - 対戦フロー時: 「この編成で対戦 ▶」→ マッチングへ直行
  - 通常時: 「対戦へ進む ▶」→ 編成を装備して対戦タイプ選択（ModeSelect）へ
  - どちらも**ログイン中は押下時にスロット自動保存**（オンライン対戦はサーバーの is_active チームを使うため、保存せず対戦に進んだ時の新旧編成の乖離を防ぐ）。保存失敗でもフローは継続
- 「プリセットチーム」はヘッダー（プリセット選択の隣）へ移動（戻り先=編成画面）

## 操作フロー

### 新規編成の場合

1. 「＋」タブをタップして新規スロットを作成
2. デフォルトシステムを6つから選択（初期配置テンプレート）
   - 4-4-2 / 3-5-2 / 3-6-1 / 4-3-3 / 4-2-3-1 / 3-4-3
3. 選んだシステムに応じて、手持ちコマから自動的にコスト1のコマが配置される
4. ここから自由に編集する

### コマの入れ替え（フィールド上のコマを別のコマに変更）

1. フィールド上のコマをタップ → 選択状態になる
   - PC版：横のスペースにコマ詳細を大きく表示（顔イラスト、キャラクター名、背景テキスト、ポジション、コスト）
   - スマホ版：コマが拡大して略称・コスト等を表示
2. 「入れ替え」アクションを選択 → カードグリッドが開く
3. カードをタップ → 即入れ替え完了、コスト表示が更新される
   - コスト16超過のカードはグレーアウトで選択不可
   - 使用中のカードは半透明で選択不可

### コマの移動（フィールド上の位置変更）

1. コマを長押し（またはドラッグ）すると移動モードに入る
2. フィールド上の好きな場所にドラッグして配置
3. ドロップすると移動完了

**制約：**
- GKも含め全コマが自由に移動可能
- コマ同士は重ならない

※操作方法（タップ＝入れ替え、長押し＝移動）は要検討。逆のほうが直感的であれば変更可。

### サブ（ベンチ）の編集

1. サブエリアの枠をタップ → 選択状態になる
2. 「入れ替え」アクションを選択 → カードグリッドが開く（コスト制限なし、全カード選択可能）
3. カードをタップ → 即配置完了

### 名前の入力

1. コマの名前表示部分をタップ
2. テキスト入力欄が表示される
3. 任意の名前を入力（選手名など）

- コマ購入後にポジションを入れ替えることで本来のシステムが完成する

## セーブ/ロード（v3: サーバー保存）

### 保存先とAPI

- 保存先: D1 `teams` テーブル（`POST/PUT /api/teams`、スロットごとに1レコード）
- `field_pieces` / `bench_pieces` は以下の要素を持つJSON配列:

```json
{
  "piece_id": 42,
  "position": "OM",
  "cost": 3,
  "col": 10,
  "row": 15
}
```

- **col/row はHEXグリッド座標**（col 0-21 / row 0-33、自陣側にスナップ）。ベンチはcol/row省略可
- **position/cost はサーバーが `piece_master` から解決する**（クライアント申告値は信用しない — コスト偽装防止）
- スロット名は `teams.name`、システムは `teams.formation_preset`
- スロットごとに独立して保存、最大10スロット（利用可能枠数はentitlement判定）
- ゲスト: 編成ドラフト（未確定の作業状態）をlocalStorageに自動保存。明示セーブはログイン誘導

## UI/UX要件

### PC版レイアウト
- フィールドを画面中央に配置
- 左右どちらかにコマ詳細パネル（選択時に顔イラスト・キャラクター名・背景テキスト・ポジション・コストを大きく表示）
- カードグリッドはモーダルまたは右パネルに表示
- セーブスロットのタブは画面上部

### スマホ版レイアウト
- フィールドが画面全体を占める
- コマをタップすると拡大表示（略称・コスト表示）
- カードグリッドは画面下部からスライドアップ（2列表示）
- セーブスロットのタブは画面上部（横スクロール）

### 共通
- フィールドはサッカーピッチ風デザイン（縦長・ライン表示）
- コマはポジションごとに色分けされた形状
- コスト枠装飾：1=無装飾, 1.5=銅枠, 2=銀枠, 2.5=金枠, 3=虹枠
- 移動中のコマは視覚的に分かりやすく（拡大、発光、影など）
- コスト超過時は数値を赤字で警告表示
- 長押し判定は0.5秒以上
- スタメンは常に11枚（外す機能なし）
