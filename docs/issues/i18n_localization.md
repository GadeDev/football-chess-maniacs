# i18n ローカライズ状況

更新日: 2026-07-10

この文書は、`i18n_playbook.md`の7フェーズに沿ったFootball Chess ManiacSの実装・翻訳レビュー・検証状況を追跡する。2026-07-10の棚卸しでは、既存基盤を維持しつつ、残っていた画面文言、対戦演出、法務導線、プリセット紹介文、バリデーション境界を補完した。

## 現在の到達点

- 対象ロケールは`ja` / `en` / `es` / `pt` / `de` / `zh-CN` / `ko`の7言語。
- ja辞書を正本とし、全辞書は647キーで同一キー集合・同一placeholder集合を持つ。
- ja値は既存表示・正本データの鏡写しを原則とし、既存の日本語表示を変えない。
- プリセット選手77件の紹介文は、安定IDの`preset.piece.{id}.summary`キーで辞書へ鏡写しした。`presetTeams.test.ts`でja辞書値と正本データの一致を検証する。
- 選手名・ポジション略称・棚識別記号`GR`は原語固定とする。`GR`も`preset.group_rank`経由で表示し、直書きには戻さない。
- `src/middleware/validation.ts`の違反理由は、人間向け文字列ではなく構造化した`code + params`を返す。shared/middleware層へクライアントi18nを持ち込まない。
- ロケール選択は`fcms.locale`へ永続化し、即時再描画に加えて`html[lang]`も同期する。ブラウザ言語検出は大小文字を正規化する。
- プリセット名は安定IDから表示時に解決し、空のセーブスロット名は言語非依存sentinelで保存する。旧7言語で保存済みの既定名も固定legacy一覧から未指定へ正規化する。

## 自動検証

2026-07-10の変更で、次を回帰テスト・監査の対象にし、最終ゲートまで完了した。

- 全7辞書のキー集合647件と`{placeholder}`集合のja基準パリティ
- `.one` / `.other`の対、およびko/zh-CNで日本語へ早期fallbackしないこと
- `LanguageSelect`の7言語/endonym、即時切替、localStorage永続化、`html[lang]`同期
- 未分類の日本語リテラル、辞書へ移したUIラベルの再直書き、モジュール直下の`t()` / `tn()`呼び出しをASTで検出
- canvas文字と実行時enum（接続状態、action、event、shoot outcome）を内部トークンのまま表示しないこと
- プリセット紹介文77件の安定キー一意性とja原文との一致
- 保存済みプリセット名とスロット既定名がロケール変更へ追従すること

最終結果は825 tests passed / 10 live E2E skipped、型チェック成功、Vite production build成功、`git diff --check`成功。ブラウザでは7言語の即時切替、`html[lang]`、英語のリロード永続化、日本語で保存したプリセット名の英語追従、390px幅での全7言語横overflowなしを確認した。ドイツ語の音量ラベルで検出した固定幅超過は、最小幅レイアウトへ修正後に再検査している。

## 画像監査

既存のラスタ画像を確認した結果、翻訳対象となるユーザー向け文章は見つからなかった。キャラクタースプライトには文言がなく、コマ画像のポジション略称・ランク表記は原語固定の識別記号として扱う。このため、今回の多言語化では画像生成による差し替えは不要と判断した。

次の欠落参照は監査中に確認したが、今回のi18n変更以前から存在する非i18n項目である。

- `/assets/board/board_mobile.png`: 盤面座標と厳密に整合する正本アセットが必要。汎用生成画像で代替しない
- `/manifest.json`: PWA/HTMLメタデータの欠落。翻訳画像とは無関係

これらは別のアセット/PWA修復タスクとして扱い、本i18n作業の完了判定には混在させない。
